# Draft Sandbox Environment

## Overview
סביבת טיוטה לוקאלית לניסויים עיצוביים תחת `app/draft/`, שנבנתה לבקשת המשתמשת ("סביבה לנסיונות עיצוביים שלא תעלה לורסל"). כל התיקייה נמצאת ב-`.gitignore` (כלל `/app/draft/`, שורה 43) כך שה-auto-commit האגרסיבי (`git add -A` → push → Vercel) לעולם לא תופס אותה — היא רצה רק ב-`next dev` המקומי ולעולם לא מגיעה ל-GitHub/ורסל. שכבת הגנה שנייה: `app/draft/layout.tsx` קורא `notFound()` אם `process.env.VERCEL` קיים, כך שגם אם קובץ טיוטה יידחף בטעות (`git add -f`) הוא יחזיר 404 בכל סביבת ורסל. תוכן: `layout.tsx` (שומר + תג צף "טיוטה · לוקאלי בלבד" + `robots:noindex`), `page.tsx` (מרכז טיוטות בעברית עם הסבר + רשימת ניסויים + הוראות הוספה), ו-`playground/page.tsx` (fork חי של `app/page.tsx` — אותן קומפוננטות + `getContent()` אמיתי, נקודת פתיחה לשחק בלי לגעת באתר). מוגש ב-`/draft` ו-`/draft/playground`. **מדיניות עבודה (מעוגנת ב-`CLAUDE.md`):** כל שינוי באתר מתחיל בטיוטה, וקידום לקבצים האמיתיים/ורסל קורה רק בהוראה מפורשת ("לדחוף לורסל" / "push to Vercel"). קידום ממומש ע"י `scripts/promote-draft.mjs` (`npm run promote:draft`): מסמנים קובץ טיוטה ב-`// @promote-to: <target>` והסקריפט מעתיק אותו למקומו האמיתי בריפו (ומסיר את שורת הסימון), משם ה-auto-commit דוחף לורסל.

## Open Questions
- מנגנון ה-auto-commit הוא חיצוני (אין git hooks בריפו — נבדק) ולא אותר בפועל; ההגנה מסתמכת על כך שהוא מכבד `.gitignore`. אומת ש-`git add -A --dry-run` לא מעלה שום קובץ טיוטה — אבל אם המנגנון אי-פעם ישתמש ב-`git add -f`/`-A -f`, שכבה 1 נעקפת (שכבה 2 — שומר ה-VERCEL — עדיין תופסת).
- ה-playground הוא כרגע העתק סטטי של הרכבת עמוד הבית; אם `app/page.tsx` ישתנה מהותית, ה-fork יתיישן ידנית (לא מסונכרן אוטומטית). מקובל — זו הנקודה של סביבת ניסוי.
